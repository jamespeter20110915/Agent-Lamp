# v0.3.0 Claude Code Support

## Goal

Support Claude Code as a first-class Agent-Lamp source.

## Initial Assessment

- Existing code already has a Claude hook entrypoint at `hooks/claude/status-hook.sh`.
- Existing status mapping supports Claude-compatible lifecycle events including `UserPromptSubmit`, `Notification`, `PermissionRequest`, `PostToolUse`, `PostToolUseFailure`, `Stop`, `SessionStart`, and subagent start/stop events.
- Existing docs include a Claude Code hook section and a `hooks/claude/settings.example.json` reference.
- Current Claude hook sends directly through `host/agent_lamp.py` unless `AGENT_LAMP_QUEUE` is set. Codex already defaults to the queue bridge.

## Gap

Claude Code support is partially present, but v0.3.0 should make it reliable and easy to install:

- Add a project-level `.claude/settings.json` example or installer path if we want one-command setup. Done: project settings now install the Agent-Lamp hooks when Claude Code is opened in this repo.
- Decide whether Claude Code should also default to the queue bridge for parity with Codex. Done: `hooks/claude/status-hook.sh` now defaults `AGENT_LAMP_QUEUE` to `/private/tmp/agent-lamp-queue.tsv`.
- Expand the Claude hook example to cover `PostToolUse`, `PreToolUse`, `SessionStart`, and `SubagentStop`. Done in both `.claude/settings.json` and `hooks/claude/settings.example.json`.
- Add a user-customizable completion sound. Done: the daemon accepts `--done-sound` or `AGENT_LAMP_DONE_SOUND`, supports macOS system sound names and custom audio file paths, and only plays on `running`/`waiting` to `ok` transitions.
- Validate behavior in a real Claude Code session with the Tab5 running over HTTP.

## Current Answer

The original failure was that Claude Code had no Agent-Lamp hook installed: the repo had no `.claude/settings.json`, and the global `~/.claude/settings.json` had no `hooks` block. Claude Code therefore never called `hooks/claude/status-hook.sh`.

The repo now installs project-level Claude Code hooks and routes them through the queue bridge. Remaining validation is a live Claude Code session from this repo while the daemon is running.

## Live Diagnosis

- A simulated Claude Code `UserPromptSubmit` event now writes `set	running	claude	Agent-Lamp	Agent is working` to `/private/tmp/agent-lamp-queue.tsv`.
- The running daemon reads that line and logs `sent: set	running	claude	Agent-Lamp	Agent is working`.
- The daemon still cannot reach the configured wireless target: `/private/tmp/agent-lamp-daemon.err` reports `Could not reach lamp at http://agent-lamp.local`.
- This means the software hook path is now wired, but the Tab5 is not reachable at `agent-lamp.local` from the Mac. The next live step is to use the IP address shown on the Tab5 screen or reconnect/flash the device so `GET /ping` responds.
